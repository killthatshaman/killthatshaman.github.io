蓝帽杯初赛writeup

2021年04月29日 900 字 大概 4 分钟 ctf pwn

蓝帽杯初赛 portable_rpg wp

arm架构的题目

漏洞

漏洞点在于add的时候如果选择的职业不为123中的一个,那么就只会malloc0x20的player chunk且不对chunk中的值进行操作,而不会去malloc name chunk
因此可以把name chunk中对应打败巨龙的count设成1,free掉之后再malloc出来,那么对应的player就可以show了.
libc版本是2.26

strings ./libc.so.6 | grep "GNU"
GNU C Library (Buildroot) stable release version 2.26, by Roland McGrath et al.
Compiled by GNU CC version 6.4.0.
        GNU Libidn by Simon Josefsson

同时因为malloc的时候不会对其中的值有操作,那么如果本来fd字段有tcache链表指针(对应name_ptr字段),那么如果再将其free就可以达成doublefree

调试

arm最麻烦的就是调试,首先上网下2.26的libc
找了半天只找到这个
然后用

rpm2cpio *.rpm| cpio -div

解包,之后把lib文件夹里面的libc-2.26.so和ld-2.26.so拖到题目文件夹下
这里有个坑点,解包出来的libc里面有libc文件和符号链接,符号链接是不能直接复制进虚拟机的,否则会直接强制关机,原因未知,可以用tar打个包再拖进去,进去解压
然后

qemu-arm -g 1234 -L ./ ./protable

如果提示找不到ld-linux-armhf.so.3,那么就mv ld-2.26.so ld-linux-armhf.so.3
libc.so.6同样,缺什么就重命名什么.
另外新开一个终端,

gdb-multiarch ./protable -q
pwndbg> set architecture arm
pwndbg> target remote localhost:1234

就可以调试了
